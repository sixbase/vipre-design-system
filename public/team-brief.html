How we all use our shared parts

The short version. We built a box of ready-made UI pieces (buttons, menus, tables). Everyone builds from the same box — so every screen looks like Vipre, and one fix reaches everyone.

Two ways to use a piece

🧰
Better

Grab it from the box

Add the box once. Then ask for a piece with one line. We improve it → everyone gets the better one, free.

✏️
Okay

Copy it by hand

Copy the code from our site. Works — but your copy goes stale when we fix the real one.

Our twist: two kinds of blocks

Some projects use React, some use Angular. We don’t want to build every piece twice. Good news — a piece has two parts:

  • ✓The look (colors, shapes) is just CSS. Already works everywhere.
  • ✓The clicks we build once so React and Angular use the same one.

So the plan is: build each piece once, everyone shares it, nobody rebuilds it. Come to the site to learn; grab the real piece from the box to use.

What we need to decide together

Which blocks does each team use — React, Angular, or both?

Do we agree to build each piece once and share it?

Who sets up the box so you install it once and update in one line?
